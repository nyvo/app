This skill guides creation of distinctive, production-grade UI for Ease. The goal: interfaces that feel intentionally designed — not AI-generated slop — while staying within the Norwegian design language.

Before writing any code, read `DESIGN_SYSTEM.md` for tokens/components and `DESIGN_LANGUAGE.md` for philosophy.

## Design Thinking

Before coding, understand the context:
- **Purpose**: What problem does this interface solve? What's the user's task?
- **Tone**: Calm, trustworthy, and direct. Design quality comes from restraint and precision, not decoration.
- **Constraints**: Follow `DESIGN_SYSTEM.md` tokens. Use existing `src/components/ui/` components. Norwegian copy rules in `COPY_STYLE_GUIDE.md`.
- **Reference**: Study `CourseOverviewTab.tsx` for the gold standard page/card pattern before building.

## Creative Within Constraints

Norwegian design achieves distinction through precision, not loudness. The creative opportunity is in:

- **Typography rhythm**: Tight `tracking-tight` on page titles, generous spacing between sections, careful weight hierarchy (`font-medium` only — never `font-semibold`)
- **Spatial generosity**: Whitespace is the design. When unsure, add more space, not less. Let content breathe.
- **Shadowless depth**: Hierarchy through borders and background contrast only. No shadows. This is distinctive — most apps lean on shadows. Ease doesn't.
- **Subtle motion**: Fast, invisible transitions (`smooth-transition` 0.2s, `ios-ease` 0.3s). Motion explains, never impresses. `pageVariants` and `tabVariants` from `@/lib/motion`.
- **Surface contrast**: Canvas (zinc-100) → Surface (white + border) → Emphasis (zinc-900). Three levels, no ambiguity.
- **Information density**: Start minimal, reveal complexity as the user goes deeper. Progressive disclosure over dense dashboards.

## What Makes It Not Slop

Generic AI UI has tells. Avoid all of these:

| AI Pattern | Ease Alternative |
|------------|-----------------|
| Purple/blue gradients on white | Zinc palette, single dark accent |
| `shadow-md` on every card | Borderless or `border border-zinc-200`, no shadows |
| `font-semibold` everywhere | `font-medium` only |
| `text-lg`/`text-xl` section headers | `text-sm font-medium` — small, confident |
| Decorative icons everywhere | Icons only when they improve scan speed or signal type |
| Rounded colorful badges | Muted status colors, `text-xs font-medium` |
| Hero sections with stock illustrations | Clean empty states with action-oriented copy |
| Cookie-cutter card grids | Section titles above cards, open layouts with subtle dividers |
| Marketing language ("Explore", "Discover") | Direct Norwegian copy ("Se", "Velg", "Opprett") |
| `text-base` body text | `text-sm` — the entire app uses 14px as the workhorse |

## Implementation Rules

1. **Use existing components**: Check `src/components/ui/` before building custom. Compose from shadcn primitives.
2. **Section titles above cards**: Never put a section header inside a card container.
3. **Cards**: `rounded-xl bg-white p-6 border border-zinc-200`. Clickable cards add `hover:bg-zinc-50/50`.
4. **Text colors**: Use `text-text-primary`, `text-text-secondary`, `text-text-tertiary` — never hardcoded `text-zinc-*`.
5. **Currency**: Always `formatKroner()` from `@/lib/utils`. Never inline `kr` formatting.
6. **Page structure**: `SidebarProvider` + `TeacherSidebar` + `main` with `bg-surface`. Use `MobileTeacherHeader` for mobile.
7. **Page headers**: `px-6 lg:px-8 pt-6 lg:pt-8 pb-0` with `font-geist text-2xl font-medium tracking-tight` for H1.
8. **Empty states**: Solid borders, flat white background, centered content — no dashed borders or gradients.
9. **Norwegian copy**: Follow `COPY_STYLE_GUIDE.md`. Sentence case. No "Vennligst". No exclamation marks.

## The Quality Bar

Before shipping, ask:
- Does this feel calm, not clever?
- Would a Norwegian user trust this interface?
- Is there anything decorative that doesn't serve comprehension?
- Would this still feel good after years of daily use?
- Does it look like it was designed by a human with taste, or generated by an AI?
