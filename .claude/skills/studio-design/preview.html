Tabs & segmented controls

two controls, two jobs — not interchangeable

Underline tabs — slicing

Each tab reveals different content — different items, different filter, different section. The count behind each tab changes.

Use for: Upcoming · Today · Past — All · Active · Draft · Archived — Settings sections

Segmented control — switching lens

Each segment shows the same content, viewed differently. Format toggle, timeframe, or sort.

Use for: List/Grid/Calendar — Day/Week/Month — Revenue/Bookings/Capacity
Litmus test: if switching options changes the count of items shown, it's underline tabs (slicing). If the count stays the same, it's a segmented control (switching lens).

Which pattern, when?

Studio defaults to undo. Confirm dialog is the second tier; type-to-confirm is rare.

Toast with undo
Default. Action happens immediately; toast shows for 8s with «Angre» link. Use for: cancel signup, archive course, delete note.
Standard confirm
When undo isn't enough — irreversible but not catastrophic. Use for: delete course (with paid signups), refund payment, send to all students.
Type-to-confirm
For catastrophic — broad scope, genuinely irreversible. Forces deliberation. Use for: delete studio, delete account, terminate subscription.

Page layout & responsiveness

Sidebar present → left-align · No sidebar → center · Never stretch past max-width

The rule — sidebar present vs absent

Two layouts, one difference: mx-auto on public, omitted on dashboard. Both have max-w-5xl caps.

Dashboard — left-aligned
empty

Public — centered

mx-auto max-w-5xl px-4 sm:px-6 lg:px-8

Why not center the dashboard? NN/g eye-tracking research: 80% of fixations occur on the left half of pages. Centering content next to a left-anchored sidebar fights this natural reading pattern. Linear, Notion, Stripe, Vercel, and Cal.com all left-align their dashboard content.

Max-widths by surface — four values

Each with a clear job. Most pages fall into the first two; the others are edge cases.
