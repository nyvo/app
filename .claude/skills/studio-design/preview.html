Controls

Buttons

Filled-only. Two filled variants — primary (sand-12) and secondary (sand-3 / bg-muted, no border). ghost is the transparent low-emphasis option (inline row actions, sidebar nav). soft is the persistent muted-circle for dedicated icon controls (close ×, kebab). outline is deprecated — migrate when you touch the file.

Variants

Destructive exists as a reserved variant. Confirmation dialogs use the monochrome primary button instead — the verb carries the destructive signal.

Icon controls — soft variant

Dedicated icon controls — close × in dialog/sheet/drawer headers, kebab menu triggers, share — use soft + size="icon". The persistent muted circle reads as "tappable target" without competing with primary actions. For dense inline row controls (table action columns where a 36px circle would crowd the layout) keep ghost — transparent at rest, hover-fills.

Sizes

Default to default. Use lg inside modals. Use cta for public/mobile primary actions. If an action feels too small for a button, make it plain text, icon-only, or put it in a menu.

States
Loading — async submit / save / payment

Spinner left of label · same dimensions as rest state · disabled + aria-busy="true" · no checkmark animation on success (toast carries the result).

Inputs

Label above. Placeholder only when format isn't obvious.

Tabs & segmented controls

two controls, two jobs — not interchangeable

Underline tabs — slicing

Each tab reveals different content — different items, different filter, different section. The count behind each tab changes.

Use for: Upcoming · Today · Past — All · Active · Draft · Archived — Settings sections

Segmented control — switching lens

Each segment shows the same content, viewed differently. Format toggle, timeframe, or sort.

Use for: List/Grid/Calendar — Day/Week/Month — Revenue/Bookings/Capacity
Litmus test: if switching options changes the count of items shown, it's underline tabs (slicing). If the count stays the same, it's a segmented control (switching lens).
