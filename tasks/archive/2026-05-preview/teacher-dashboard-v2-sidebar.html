What's different from the top-nav variant

  • Sidebar 232px wide, same off-white as page background — feels integrated, not "panel-y."
  • Two nav groups: primary (Hjem · Kurs · Påmeldinger · Kalender · Studio), separated by a 1px line + label, then "Innstillinger" group (Lokasjoner · Betalinger). Nav can hold more items here than the top nav, which only fit 5 + a user menu.
  • Icons on every nav item, stroke-1.75 lucide style — matches the dashboard icon convention. They're informational not decorative — could be removed for an even calmer look but feel useful at this size.
  • User pill at bottom of sidebar with name + email + dropdown chevron. Click opens the same menu the top-nav avatar would (Profil, Innstillinger, Logg ut, Vis offentlig side, etc.).
  • Page max-width 1180px (vs 1240px in the top-nav version) since the sidebar already takes 232px — keeps total layout under ~1500px on the widest screens.
  • Active state pill same quiet grey as the top-nav active state — visually consistent if we ever ran a hybrid (sidebar on dashboard, top nav on landing).

A vs B — the actual question

  • Top nav reads more "consumer product" — closer to SpareBank-1 / Vipps. Calmer at the top of the page. Limits us to 5-7 nav slots; user menu absorbs the rest.
  • Sidebar (this file) reads more "SaaS dashboard" but a calm SaaS dashboard. Holds more items visibly (we can show Lokasjoner + Betalinger as first-class nav, not buried in a user menu). Users always see where they are without dropping a menu.
  • The calm-register thesis tilts toward top nav. The "yoga teachers go to ~5 places" pattern also tilts toward top nav (sidebar's main advantage — 10+ items — isn't needed). The argument for sidebar is muscle-memory: every dashboard tool the teacher has used (Stripe, Vipps Bedrift, accounting software) probably has one.
  • If we go top nav and Lokasjoner/Betalinger feel buried in the user menu, that's the case the sidebar would have caught. If we go sidebar and the dashboard still feels "tool-y," that's the case top nav would have caught.
